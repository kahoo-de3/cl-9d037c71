# C/O チェックリスト

医療施設×医師ごとの C / O タップ漏れをスマホで確認するチェックリストアプリ。
約6か月ごとに名簿（Excel）が差し替わる前提で、**Excelを渡すだけで同じデザイン・仕様のまま再生成できる**ようになっている。

- **公開URL**: https://kahoo-de3.github.io/cl-9d037c71/
- **リポジトリ**: kahoo-de3/cl-9d037c71（GitHub Pages: main ブランチ /docs 配信）
- 検索避け: ランダムなリポジトリ名 + `noindex` メタ + robots.txt

## 名簿の差し替え手順（6か月ごと）

Claude に新しい Excel を渡して「**前回と同じC/Oチェックリストを作って**」と依頼するだけ。
Claude（または手動）が行うことは以下のみ:

```
pip install openpyxl cryptography   # 初回のみ
python tools/generate.py <新しいBook.xlsx> <PIN>
git add docs/index.html && git commit -m "update roster" && git push
gh api -X POST repos/kahoo-de3/cl-9d037c71/pages/builds   # Pagesの自動ビルドが走らないことがあるため必ず実行
```

- PIN は公開リポジトリに書かない（依頼時に指定。通常は前回と同じ）
- URL・デザイン・操作感は変わらず、**各端末のチェック履歴もそのまま残る**
- PIN を変えなければ端末側の再入力も不要

### Excel の前提レイアウト

- **1枚目のシート＝データ１、2枚目のシート＝データ２**（2枚目が無ければデータ１のみ）
- 各シートは以下の列構成。ヘッダー行（B列が「施設名」の行）を自動検出し、その次の行から読む
  ので、月セルの有無などレイアウトの細部が違っても動く

| 列 | 内容 |
|---|---|
| B列 | 施設名 |
| C列 | 氏名 |
| D列 | 値があれば（"C" でも "1" でも可）Cボタンを表示 |
| E列 | 値があれば（"O" でも "1" でも可）Oボタンを表示 |

- データ１・データ２はアプリのタイトル右「⇄」ボタンで切り替え。チェック状態は
  データセットごとに独立保存（データ１=`co-check-YYYY-M`、データ２=`co-check-d2-YYYY-M`）

- 行数は可変（B列に値がある行をすべて取り込む）
- **同じ施設×同じ氏名の行は1行に統合**される（元Excelには同じ名簿が2ブロック
  入っていることがあるため。C/Oフラグは論理和、保存キーは初出の行番号）
- 外字（私用領域文字）は `tools/generate.py` の `PUA_MAP` で通常漢字に自動置換
  （例: 「祐」U+E682 → 祐）。未知の外字は生成時に警告が出るので置換表に追加する
- **表示は施設名の読みで五十音順にソート**される（Excelの行順ではない）。
  読みは generate.py の `YOMI` 辞書（施設名→ひらがな）で管理。新しい施設が来ると
  生成時に警告が出るので、正しい読みを調べて辞書に追加する。
  同じ施設が複数行ある場合はExcelの行番号順に隣り合って並ぶ（行番号の表示はしない）

## アプリ仕様

### 画面・操作
- **PINロック**: 起動時にPIN入力。最後の桁を入力すると自動で解錠（ボタン操作不要）。
  一度解錠した端末は次回から自動で開く（鍵をlocalStorageに保持）
- **年・月プルダウン**: 2026〜2030年 / 1〜12月。年月ごとに独立したチェック状態を保持
- **C / O ボタン**: 行ごとに存在する品目だけ表示（C のみ・O のみ・両方の3パターン）。
  タップでON/OFFトグル
- **行の状態表示**
  - 未タップ: 白背景
  - 一部タップ: 薄い灰色背景・灰色文字
  - 必要な品目すべてタップ（完了）: 濃いグレー背景・白系文字・取り消し線
- **進捗バー + 「完了 x / 総数」** をヘッダーに常時表示
- **未完了のみ表示**: 完了行を隠すフィルタ。全件完了なら「この月は全件チェック済みです🎉」
- **リセット**: 表示中の年月のチェックを全消去（確認ダイアログあり）
- バックアップ/復元ボタンは**なし**（過去にあったが要望により削除）

### デザイン
- 背景は**常に白**（ダークモードでも白固定、`color-scheme: light`）
- アクセント色: ティール `#0B7285`／完了グレー `#66666A`／部分グレー `#E9E9E6`
- フォント: システム日本語フォント（Hiragino Sans / Yu Gothic UI / Meiryo）
- モバイルファースト（4:3等の指定なし）。タップ目標は46px角

### データ保護
- 施設名・氏名は **AES-GCM で暗号化してHTMLに埋め込み**（平文はページにもリポジトリにも存在しない）
- 鍵はPINから PBKDF2-SHA256（310,000回）で導出。PINはどこにも保存されない
- チェック状態（行番号とON/OFFのみ）は端末のlocalStorageに平文保存（個人情報を含まないため）

## 実装上の注意（ハマりどころの記録）

1. **claude.aiアーティファクトでは永続化不可**: サンドボックスが毎回作り直され
   localStorageが消える。恒久利用は必ずGitHub Pages版で
2. **iOS Safariのロック画面**: `position:fixed` オーバーレイはキーボード表示中に
   見た目と当たり判定がずれてボタンが押せなくなる → ロック画面は通常フロー配置にし、
   最終桁入力での自動解錠を実装済み
3. **hidden属性はCSSのdisplay指定に負ける**: `.row{display:flex}` があると
   `hidden` が効かない → `[hidden]{display:none !important}` を入れてある（消さない）
4. **GitHub Pagesはpush後に自動ビルドが走らないことがある** →
   `gh api -X POST repos/kahoo-de3/cl-9d037c71/pages/builds` で必ず手動トリガーし、
   `builds/latest` のcommitがHEADと一致するまで確認する
5. **Excelの文字コード**: セルに外字（U+E000〜U+F8FF）が混ざることがある。
   generate.py が警告を出すので `PUA_MAP` に追加する
6. 生成のたびに salt/iv が変わるため、名簿が同じでも docs/index.html は毎回差分が出る（正常）
